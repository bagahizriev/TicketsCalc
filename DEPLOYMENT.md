# Инструкция по деплою на GitHub Pages

## Шаг 1: Создание репозитория

1. Создайте новый репозиторий на GitHub с именем `TicketsCalc`
2. Не добавляйте README, .gitignore или лицензию (они уже есть в проекте)

## Шаг 2: Загрузка кода

```bash
# Инициализация git (если еще не сделано)
git init

# Добавление всех файлов
git add .

# Первый коммит
git commit -m "Initial commit: PWA калькулятор билетов"

# Переименование ветки в main
git branch -M main

# Добавление remote (замените YOUR_USERNAME на ваш username)
git remote add origin https://github.com/YOUR_USERNAME/TicketsCalc.git

# Отправка кода
git push -u origin main
```

## Шаг 3: Настройка GitHub Pages

1. Перейдите в Settings вашего репозитория
2. В левом меню выберите "Pages"
3. В разделе "Source" выберите "GitHub Actions"
4. Сохраните изменения

## Шаг 4: Проверка деплоя

1. Перейдите во вкладку "Actions" в вашем репозитории
2. Дождитесь завершения workflow "Deploy to GitHub Pages"
3. После успешного деплоя ваше приложение будет доступно по адресу:
   `https://YOUR_USERNAME.github.io/TicketsCalc/`

## Автоматический деплой

После настройки каждый push в ветку `main` будет автоматически запускать деплой.

## Локальная разработка

```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm run dev

# Открыть http://localhost:3000
```

## Сборка для продакшена

```bash
# Сборка и экспорт статических файлов
npm run build

# Файлы будут в папке /out
```

## Генерация PNG иконок (опционально)

Если хотите использовать PNG вместо SVG:

1. Откройте `public/create-icons.html` в браузере
2. Скачайте сгенерированные иконки
3. Сохраните их как `public/icon-192.png` и `public/icon-512.png`
4. Обновите `public/manifest.json` для использования PNG иконок

## Проблемы и решения

### Workflow не запускается

- Убедитесь, что в Settings → Actions → General разрешены GitHub Actions
- Проверьте, что файл `.github/workflows/deploy.yml` существует

### 404 ошибка после деплоя

- Проверьте, что в `next.config.js` правильно указан `basePath: '/TicketsCalc'`
- Убедитесь, что имя репозитория совпадает с basePath

### PWA не устанавливается

- Проверьте, что сайт работает по HTTPS (GitHub Pages автоматически использует HTTPS)
- Откройте DevTools → Application → Manifest и проверьте наличие ошибок
- Проверьте, что Service Worker зарегистрирован в DevTools → Application → Service Workers

## Тестирование PWA

### На iOS (Safari)

1. Откройте сайт в Safari
2. Нажмите кнопку "Поделиться"
3. Выберите "На экран «Домой»"
4. Приложение появится на домашнем экране

### На Android (Chrome)

1. Откройте сайт в Chrome
2. Нажмите на три точки в правом верхнем углу
3. Выберите "Установить приложение"
4. Приложение появится в списке приложений

### На Desktop (Chrome/Edge)

1. Откройте сайт
2. В адресной строке появится иконка установки
3. Нажмите на нее и подтвердите установку

## Обновление приложения

После внесения изменений:

```bash
git add .
git commit -m "Описание изменений"
git push
```

Workflow автоматически задеплоит новую версию.
